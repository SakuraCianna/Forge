# Forge

[中文](README.md) | [English](README.en.md)

Forge is an open-source local AI coding agent desktop app. It is not a VS Code fork and not an editor plugin marketplace. It is a project-oriented agent workbench where you choose a local repository, configure your own models, generate plans, review file diffs, run verification commands, and commit changes after confirmation.

The goal is to move AI coding from suggestions inside a chat box to a local engineering workflow that is reviewable, recoverable, and verifiable.

## Features

### Local Project Workbench

- Open a local project directory and build a text-file index.
- Browse project files, preview content, and review diffs.
- The project file tree shows non-sensitive files even when they are ignored by `.gitignore`; agent directory, search, and glob tools still respect `.gitignore` to keep automated search scoped and responsive.
- Large file trees use lazy directory loading and paged large directories.
- Project scan metadata is cached in the local app data directory.
- File changes generated by the agent enter a review queue before they are written to disk.

### Agent Task Threads

- Each request is kept in a task thread with the user prompt, model plan, execution log, file events, and command results.
- Plans are parsed into agent actions such as file reads, directory listing, glob, text search, edits, commands, and Git checks.
- The confirmation queue collects pending diffs, command approvals, manual gates, commit gates, and recovery steps.
- Agent profiles control permissions, planning limits, verification behavior, and recovery behavior.
- Stop pauses the agent queue without skipping review or command gates.

### Attachments

- The composer supports attachments through the add button, drag-and-drop, and paste.
- Images, PDFs, DOCX, XLSX, CSV, TSV, Markdown, JSON, code files, and common text files are parsed locally where possible.
- Sensitive attachments such as `.env`, private keys, certificates, and databases are skipped by default.

### Plugins And Skills

- The sidebar includes a plugin directory. A plugin organizes a group of reusable skills.
- Forge scans common local skill locations on startup, including `~/.codex/skills`, `~/.agents/skills`, and `SKILL.md` files inside the Codex plugin cache.
- The plugin page can switch between a plugin list and a skill list. Plugin details show bundled skills; skill details show source, local file path, and core files.
- Core files from locally scanned skills can be previewed read-only in the skill details page, making `SKILL.md` and related resource content inspectable in the UI.
- The plugin page includes a GitHub extension entry. Users can open repositories in `https://github.com/owner/repo` or `owner/repo` format and manually inspect or download extensions. Forge does not automatically clone, install, or execute third-party repository code in the current version.
- The composer add menu can insert plugins or skills into the current task.
- Typing `/` opens Forge commands and skill candidates. Commands run UI operations such as status, model settings, or feedback instead of being sent as normal chat text.
- Typing `@` searches files, plugins, and skills.
- Added plugin, skill, and file references are sent as context for the current request. They do not grant external service authorization or execute third-party plugin actions.

### Models And Providers

Forge includes provider presets for OpenAI, Anthropic, Gemini, OpenRouter, DeepSeek, Kimi, DashScope, Z.AI, MiniMax, SiliconFlow, Volcengine, Qianfan, Hunyuan, Groq, Together AI, Mistral AI, xAI, Fireworks AI, Cerebras, StepFun, ModelScope, Xiaomi MiMo, GitHub Models / Copilot, and Ollama.

You can also add custom OpenAI-compatible API providers with their own base URL, API key, model list, and pricing.

### Commands, Git, And Verification

- Run controlled PowerShell commands inside the current project directory.
- Stream command output into the task thread.
- Cancel running commands.
- Configure command safety rules.
- View Git status and diff summaries.
- Create commits after entering a commit message.
- Create persistent Git worktrees from the project menu.
- Forge does not push automatically.

### Personalization And Usage

- Built-in development, review, and documentation agent profiles.
- Project-scoped agent memories.
- Response style, custom instructions, background image, and UI language settings.
- Local usage and cost estimates based on recorded token usage and configured rates.

## FAQ

### Does Forge execute downloaded plugins automatically?

No. The current GitHub extension entry opens repositories for user review and manual download only. Forge does not automatically clone, install, or execute third-party extension code.

### Where are local skills discovered?

Forge scans `~/.codex/skills`, `~/.agents/skills`, and `SKILL.md` files inside the Codex plugin cache.

### Does Forge upload my project files?

Forge reads local files to build context for the model requests you initiate. Sensitive files such as `.env`, private keys, certificates, and databases are excluded by default.

### Does Forge push Git commits?

No. Forge can create local commits after explicit confirmation, but it does not push automatically.
