# Forge 性能优化体系

本文档记录 Forge 当前已经落地的性能策略和后续路线。这里不把规划写成已完成能力。

## 已落地策略

- 项目文件树展示所有非敏感文件, 不再因为 `.gitignore` 隐藏普通文件, 但 `.env`, `.git`, 私钥和凭据路径仍由安全策略拦截。
- 文件页使用按目录懒加载: 初始只请求根目录, 展开文件夹时再加载该目录的直接子项, 避免一次性渲染整棵大项目文件树。
- 单个目录如果条目很多, 文件树会按页请求并提供“加载更多”, 避免一个超大目录一次性占满渲染内存和主线程时间。
- 目录分页结果中的文件大小会在当前页内并发读取, 减少展开大目录时逐个等待文件元数据的延迟。
- 文件树懒加载仍支持深层文件自动展开: 当用户直接预览某个深层文件时, Forge 会在后台补齐父目录链。
- 项目扫描结果会持久化到本地应用数据目录的 `project-indexes` 缓存中, 后续扫描可用旧索引复用未变化文件的元数据。
- 同一路径的并发项目扫描会在主进程复用同一个进行中的扫描任务, 最近项目索引也会保留在内存中作为下一次扫描的增量参考。
- Agent 内置项目级工具也会复用同一项目扫描缓存协调层, 避免工具链绕过主窗口缓存后重复扫描大项目。
- 项目扫描读取 `AGENTS.md`, `.github/copilot-instructions.md` 和 `.cursor/rules/*` 等指令文件时会复用未变化文件的内容快照, 文件修改或删除后按元数据自动失效。
- 扫描元数据包含文件大小和修改时间, 为后续全文索引和向量索引判断增量更新范围打基础。
- Agent 的目录列表, 文本搜索和 glob 工具默认仍遵循 `.gitignore`, 并保留结果数量和文件大小上限, 防止自动化工具在大项目里无限搜索。
- `.gitignore` 规则会在主进程编译为有界内存缓存, 同一路径的并发创建会复用同一个 matcher, 文件修改或删除后会按元数据自动失效。
- 受控文本搜索已经接入本地全文索引 v2: 先在内存和应用数据目录的 `project-text-indexes` 中复用未变化文件的行级快照, 再用内存倒排候选索引缩小 ASCII 关键词查询范围; 中文和符号类查询会自动回退到精确逐行匹配。
- `project-text-indexes` 持久化缓存有条目数和字符数预算, 缓存只作为重启后的加速层; 超出预算的文件会在下一次搜索时按需重新读取, 避免大项目把应用数据目录和启动内存拖垮。
- `project-text-indexes` 会按最近使用保留有限数量的缓存文件, 跨项目长期使用时自动清理旧缓存, 防止应用数据目录持续膨胀。
- 文件预览对文本和媒体大小有上限, 大文件返回不可内联预览的状态, 避免一次性读入过多内容。
- 附件预处理在本地 Worker 中执行 OCR, PDF, DOCX 和 XLSX 解析, 避免阻塞主界面。
- 模型提示词中的项目文件列表会按速度档位截断, 大项目不会把完整文件清单塞进单次 LLM 请求。

## 后续路线

- 增量扫描 v2: 基于目录批次和文件修改时间刷新局部目录, 避免每次切换项目都完整递归。
- 全文索引 v3: 在行级快照和内存倒排候选稳定后, 再评估 SQLite FTS 或更完整的持久化倒排索引, 进一步提升大项目关键词查询速度。
- 语义检索: 在全文索引稳定后, 再接入可选向量索引, 对代码块和文档摘要做本地语义召回。
- 大文件分块: 对超大文本, diff 和预览采用按块读取, 只渲染当前视窗附近内容。
- 文件树虚拟滚动: 当展开目录产生大量可见节点时, 前端只渲染视口内行。
