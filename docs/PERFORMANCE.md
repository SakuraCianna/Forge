# Forge 性能优化体系

本文档记录 Forge 当前已经落地的性能策略和后续路线。这里不把规划写成已完成能力。

## 已落地策略

- 项目文件树展示所有非敏感文件, 不再因为 `.gitignore` 隐藏普通文件, 但 `.env`, `.git`, 私钥和凭据路径仍由安全策略拦截。
- 文件页使用按目录懒加载: 初始只请求根目录, 展开文件夹时再加载该目录的直接子项, 避免一次性渲染整棵大项目文件树。
- 文件树懒加载仍支持深层文件自动展开: 当用户直接预览某个深层文件时, Forge 会在后台补齐父目录链。
- 项目扫描结果会持久化到本地应用数据目录的 `project-indexes` 缓存中, 后续扫描可用旧索引复用未变化文件的元数据。
- 扫描元数据包含文件大小和修改时间, 为后续全文索引和向量索引判断增量更新范围打基础。
- Agent 的目录列表, 文本搜索和 glob 工具默认仍遵循 `.gitignore`, 并保留结果数量和文件大小上限, 防止自动化工具在大项目里无限搜索。
- 文件预览对文本和媒体大小有上限, 大文件返回不可内联预览的状态, 避免一次性读入过多内容。
- 附件预处理在本地 Worker 中执行 OCR, PDF, DOCX 和 XLSX 解析, 避免阻塞主界面。
- 模型提示词中的项目文件列表会按速度档位截断, 大项目不会把完整文件清单塞进单次 LLM 请求。

## 后续路线

- 增量扫描 v2: 基于目录批次和文件修改时间刷新局部目录, 避免每次切换项目都完整递归。
- 本地全文索引: 先用轻量倒排索引或 SQLite FTS 提升 grep/search 速度。
- 语义检索: 在全文索引稳定后, 再接入可选向量索引, 对代码块和文档摘要做本地语义召回。
- 大文件分块: 对超大文本, diff 和预览采用按块读取, 只渲染当前视窗附近内容。
- 文件树虚拟滚动: 当展开目录产生大量可见节点时, 前端只渲染视口内行。
